Enabling Use test data injects one of four different types of test data, permitting all possible GUI states to be visualized, instead of calling the weather service. The test result types are listed below. Each time search is clicked, the next result type is used. The first state shown is NoResult.

  • No Result - default state, or if user clicks Search with no city name. In this state, the result component is invisible.
  • ApiSuccessNoData - Search API succeeds, but no city matching the specified name is found
  • ApiFailed - the Search API returns an error
  • ApiSuccessWithData - the search API succeeded and returned a result

When Use test data is check, simulated test data is displayed. Uncheck this box to call the weather service..
